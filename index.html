🔧 Setup — Get Your Google Client ID

To use this app, you need a free Google OAuth Client ID. Here's how:

  1. Go to Google Cloud Console → Credentials
  2. Create a project (or select existing)
  3. Click "+ CREATE CREDENTIALS" → "OAuth client ID"
  4. Application type: "Web application"
  5. Under "Authorized JavaScript origins", add your URL (e.g. http://localhost:8080)
  6. Click "Create" and copy your Client ID
  7. Go to API Library → Gmail API and click "Enable"
  8. Paste the Client ID in the field at the top of this app

What is Gmail Lab?

Gmail Lab lets you explore your Gmail inbox — search, read, and learn Gmail search syntax in a fun way!

How do I connect?

Enter your Google OAuth Client ID, then click "Sign in with Google". You need a free Google Cloud project.

Is it safe?

Yes! The app uses read-only access — it cannot modify, delete, or send emails. Everything stays in your browser.

Can anyone use it?

Yes! Each user signs in with their own Google account. No data is shared between users.

🔑 Getting Your Google Client ID

  1. Go to Google Cloud Console
  2. Click Select a project → New Project → name it "Gmail Lab" → Create
  3. Click "+ CREATE CREDENTIALS" → "OAuth client ID"
  4. If asked, configure the OAuth consent screen first:
    • User type: External → Create
    • App name: "Gmail Lab", add your email
    • Scopes: skip (click Save)
    • Test users: add your own Gmail address
    • Click Back to Dashboard
  5. Now create credentials: Application type = Web application
  6. Under Authorized JavaScript origins, add:
    • http://localhost:8080 (for local testing)
    • https://abourdim.github.io (for GitHub Pages)
  7. Click Create → copy the Client ID
  8. Go to API Library → Gmail API → click Enable

📧 How It Works

Gmail Lab uses Google's official OAuth2 + Gmail API. Your browser talks directly to Google — no server in between. The app only requests read-only access (gmail.readonly), meaning it can never modify, delete, or send emails.

⚙️ Google OAuth Configuration Guide

What is OAuth? OAuth lets users sign in with their Google account and grant your app limited access (read-only Gmail) without sharing their password.

Step 1: Create a Google Cloud Project

  1. Go to console.cloud.google.com
  2. Click the project dropdown (top bar) → New Project
  3. Name it (e.g. "Gmail Lab") → Create
  4. Make sure this project is selected in the top bar

Step 2: Enable Gmail API

  1. Go to API Library → Gmail API
  2. Click Enable
  3. Wait for it to activate (takes a few seconds)

Step 3: Configure OAuth Consent Screen

  1. Go to Google Auth Platform → Audience
  2. User type: External (unless you have Google Workspace)
  3. Go to Branding: App name = "Gmail Lab", add your email
  4. Scopes: skip (the app requests scopes at runtime)
  5. Audience → Test users: click + Add users → add your Gmail address

Without adding yourself as a test user, you'll get "Access blocked" when signing in.

Step 4: Create OAuth Client ID

  1. Go to Clients (left sidebar) or Credentials
  2. Click + CREATE CREDENTIALS → OAuth client ID
  3. Application type: Web application
  4. Name: "Gmail Lab"
  5. Authorized JavaScript origins — add ALL URLs where the app runs:
    • http://localhost:8080 (local dev)
    • http://localhost:8765 (if using different port)
    • https://yourusername.github.io (GitHub Pages)
  6. Leave Redirect URIs empty (not needed for this app)
  7. Click Create → copy the Client ID

The Client ID looks like: 123456789-abc.apps.googleusercontent.com

Step 5: Use It

  1. Paste the Client ID in the Gmail Lab app
  2. Click Sign in with Google
  3. On the "Google hasn't verified this app" screen, click Continue (safe — you're the developer)
  4. Grant read-only Gmail access

⚠️ Common Pitfalls & Fixes

1. "Access blocked: app not verified"

Cause: Your Gmail is not in the test users list.
Fix: Google Auth Platform → Audience → Test users → + Add users → add your email.
Note: You can add up to 100 test users. Each must be a real Google account.

2. "Google n'a pas validé cette application"

Cause: Normal for apps in testing mode.
Fix: Click "Continuer" (or "Continue" in English). This is safe since you are the developer.
To remove this screen: Publish the app (Audience → "Publier l'application"), but this requires Google verification for sensitive scopes like Gmail.

3. "Error 400: redirect_uri_mismatch"

Cause: Your current URL is not in the Authorized JavaScript origins.
Fix: Go to Credentials → edit your OAuth client → add the exact URL (including port). No trailing slash!
Example: https://abourdim.github.io (not https://abourdim.github.io/)
Wait: Changes can take 5 minutes to propagate.

4. "Error 403: access_denied"

Cause: Gmail API is not enabled, or the user is not a test user.
Fix: Enable Gmail API AND add the user to test users.

5. "idpiframe_initialization_failed"

Cause: Third-party cookies are blocked in your browser.
Fix: Allow third-party cookies for accounts.google.com, or try in a different browser. Incognito mode often blocks these.

6. Token expires after 1 hour

Cause: Google access tokens have a 1-hour lifetime. This is by design.
Fix: The app attempts auto-refresh. If it fails, click Sign In again. No data is lost.

7. "popup_closed_by_user"

Cause: The Google sign-in popup was closed before completing.
Fix: Click Sign In again and complete the full flow. Don't close the popup.

8. App works locally but not on GitHub Pages

Cause: Missing origin in OAuth credentials.
Fix: Add https://yourusername.github.io to Authorized JavaScript origins. Note: use the root domain, not the full path.
Also check: GitHub Pages may take a few minutes to deploy after a push.

9. "This app is blocked" (Workspace accounts)

Cause: Google Workspace admin has restricted third-party app access.
Fix: Use a personal Gmail account, or ask your Workspace admin to allow the app. Alternatively, set the OAuth app type to "Internal" (Workspace only).

10. Emails show HTML entities (' etc.)

Cause: Some email snippets contain encoded characters.
Fix: This is a display issue from Google's API. The full email content (click to read) should render correctly.

🔒 Privacy & Security

Read-only — the app cannot modify, delete, or send emails.

No server — all API calls go directly from your browser to Google.

No tracking — no analytics, no cookies, no data stored.

Revocable — sign out anytime to revoke the access token.

Open source — inspect the code yourself on GitHub.
